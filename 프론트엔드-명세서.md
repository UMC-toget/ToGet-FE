## 1. 프로젝트 소개

ToGet은 생일·졸업·집들이 등의 선물을 여러 사람이 함께 준비하는 선물 펀딩 서비스입니다.
이번 단계에서는 **온보딩(스플래시/로그인/회원가입) → 홈(선물 둘러보기) → 마이페이지(내 정보 관리)** 의 핵심 사용자 흐름을 구현했습니다.
상품·회원 정보는 Mock 데이터로 렌더링하며, API 연동 지점은 코드 내 `// TODO` 주석으로 표기되어 있습니다.

## 3. 라우팅 구조

`src/main.tsx` → `BrowserRouter` + `AuthProvider`로 앱을 감싸고, `src/App.tsx`에서 `Routes`/`Route`로 경로를 정의합니다.

```tsx
// src/App.tsx
<Routes>
  <Route path="/"               element={<SplashPage />} />
  <Route path="/login"          element={<LoginPage />} />
  <Route path="/signup/profile" element={<ProfileSetupPage />} />
  <Route path="/home"           element={<HomePage />} />
  <Route path="/my"             element={<MyPage />} />
  <Route path="/my/profile"     element={<ProfileEditPage />} />
</Routes>
```

```tsx
// src/main.tsx
<BrowserRouter>
  <AuthProvider>
    <App />
  </AuthProvider>
</BrowserRouter>
```

> 라우팅 방식: `BrowserRouter`(History API) 기반. SPA 새로고침 대응을 위해 배포 환경(Vercel)에서 SPA fallback 처리가 필요합니다.

## 8. 상태 관리 방식

| 범위 | 방식 | 구현 |
| --- | --- | --- |
| 전역 (로그인 상태) | **Context API** | `contexts/AuthProvider.tsx` — `isLoggedIn`, `login()`, `logout()` 제공. `hooks/useAuth.ts`로 소비 |
| 지역 (UI 상태) | **useState** | 닉네임 입력, 바텀시트/모달 열림, 카테고리·가격 필터 선택, 위시 토글, 토스트 표시 등 |
| 화면 간 데이터 전달 | **Router state** | `navigate('/my', { state: { toast: '...' } })` 로 토스트 메시지 전달 |
| 파생 상태 | 렌더 중 계산 | 가격 필터링된 상품 목록, 약관 전체 동의 여부(`allAgreed`/`requiredAgreed`) 등 |

**AuthProvider (실제 코드)**

```tsx
// contexts/AuthProvider.tsx
export default function AuthProvider({ children }: { children: ReactNode }) {
  const [isLoggedIn, setIsLoggedIn] = useState(true) // TODO: 인증 API 연동 시 초기값 false로 변경
  const value = useMemo(() => ({
    isLoggedIn,
    login: () => setIsLoggedIn(true),
    logout: () => setIsLoggedIn(false),
  }), [isLoggedIn])
  return <AuthContext.Provider value={value}>{children}</AuthContext.Provider>
}
```

> ⚠️ 현재 로그인 초기값이 `true`(개발 편의용)로 설정되어 있어, API 연동 시 `false`로 변경 + 토큰 저장/갱신 로직으로 교체가 필요합니다. (코드 TODO 명시)
> 📌 문서/기획상 언급되던 **TanStack Query·Zustand는 `dev` 브랜치에 아직 미도입**입니다. 서버 상태(TanStack Query)·전역 클라이언트 상태(Zustand) 도입 시 본 항목 갱신 필요.

## 9. 트러블 슈팅 기록

> 아래 항목은 실제 코드에 반영된 해결 사례입니다. (양식은 팀 트러블 슈팅 템플릿 기준)

### TS-01. 한글 입력 시 글자수 제한(maxLength) 초과 입력

| 항목 | 내용 |
| --- | --- |
| 문제 상황 | `TextField`에 `maxLength`를 지정해도 한글 조합(IME) 입력 중에는 브라우저가 제한을 강제하지 않아 최대 글자수를 초과 입력됨 |
| 발생 시점 | 프로필 설정 / 내 정보 수정 닉네임 입력 |
| 담당자 | 이홍연 |
| 원인 | IME 조합(composition) 중에는 `maxLength` 속성이 적용되지 않는 브라우저 동작 |
| 해결 방법 | `onChange` 핸들러에서 `value.length > maxLength`인 경우 `slice(0, maxLength)`로 직접 잘라냄 |
| 참고 자료 | `components/common/TextField.tsx` |

### TS-02. 토스트가 새로고침 시 다시 노출되는 문제

| 항목 | 내용 |
| --- | --- |
| 문제 상황 | 로그아웃/계정삭제 후 마이페이지로 이동하며 토스트를 띄우는데, 새로고침 시 토스트가 재노출될 수 있음 |
| 발생 시점 | 내 정보 수정 → 로그아웃/계정삭제 완료 이동 |
| 담당자 | 이홍연 |
| 원인 | 토스트 메시지를 `navigate` state로 전달하는데, state가 히스토리에 남아 재사용됨 |
| 해결 방법 | 마운트 시 메시지를 읽은 뒤 `navigate(pathname, { replace: true, state: null })`로 state를 즉시 소거하고 일정 시간 후 숨김 처리 |
| 참고 자료 | `pages/my/MyPage.tsx` |
